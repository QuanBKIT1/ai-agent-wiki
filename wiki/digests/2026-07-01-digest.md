### ⭐⭐⭐⭐⭐ Production-Grade AI Agents for Financial Compliance: Lessons from Stripe
- **Tác giả**: Christopher Phillippi, Chrissie Cui, Hasan Tariq, Mohan Musti (AWS + Stripe)
- **Ngày đăng**: 2026-06-26
- **Thời gian đọc**: ~12 phút
- **Nguồn**: AWS Machine Learning Blog
- **URL**: https://aws.amazon.com/blogs/machine-learning/production-grade-ai-agents-for-financial-compliance-lessons-from-stripe/
- **Wiki**: [[summaries/stripe-financial-compliance-agents]] · concept [[concepts/agent-service-architecture]] · entity [[entities/stripe]]

**Tóm tắt 5 điểm chính:**
1. **Agent cần microservice riêng, không tái dùng ML inference cũ** — agent là *network-bound* (chờ foundation model + tool call), latency bất định theo số vòng ReAct, cần API stateful. Stripe tách **Agent Service** riêng, mở rộng từ vài agent lên **>100 agent trong <1 năm**.
2. **LLM Proxy microservice** — một điểm truy cập chuẩn hóa mọi foundation model (Amazon Bedrock + khác): noisy-neighbor protection, unified API, model fallback, monitoring; đổi model chỉ cần đổi 1 argument.
3. **ReAct như closed-loop control** — observation bị inject bắt buộc trở lại context, chống reasoning drift và tạo audit trace `tool → observation → reasoning` cho compliance.
4. **Task decomposition bằng DAG** — chia review phức tạp thành sub-task nhỏ ("rail" đã đo chất lượng), phụ thuộc lẫn nhau; **prompt caching giảm 60% chi phí** input token.
5. **Human giữ quyết định cuối (HITL bắt buộc)** — agent chỉ cung cấp thông tin bổ trợ; reviewer con người trả lời từng sub-task. Kết quả: **-26% median review handling time**, **>96% helpfulness**, full audit trail đạt chuẩn examination.

## Xu hướng nổi bật tuần này

- **Infrastructure-first thắng model-first**: điệp khúc lặp lại từ nhiều nguồn (MLflow, Harness Engineering, và giờ là Stripe) — reliability đến từ harness/hạ tầng (microservice riêng, cost instrumentation, audit log, rail hẹp), không phải từ prompt hay model tốt hơn.
- **Agent = distributed systems engineering**: agent network-bound cần tách khỏi cluster ML compute-bound; DAG orchestration + async trở thành mặc định cho review phức tạp.
- **HITL vẫn là điều kiện cho ngành regulated**: autonomy tối đa không phải mục tiêu; ROI đến từ việc agent *hỗ trợ* con người + audit trail.
